# Specs

이 폴더는 견적 시스템의 계산, 입력, 검수, 변경 원칙을 문서로 고정하는 곳입니다.

## 최상위 원칙

### 1. 같은 입력이면 같은 결과

견적 시스템은 재현 가능해야 합니다.  
따라서 최종 금액 계산은 결정론적 로직으로만 처리합니다.

### 2. 비정형 문서 자동 반영 금지

PDF, Word, 현장조사서, 계약서 같은 비정형 문서는 자동으로 금액에 반영하지 않습니다.

허용 범위:

- 후보 항목 추출
- 검토 메모 생성
- 구조화 입력 초안 제시

금지 범위:

- AI가 읽은 문장을 바로 금액으로 반영
- 사용자의 확인 없이 수량/단가 자동 확정
- 계산 엔진에 비정형 원문을 직접 투입

### 3. 문서 우선 변경

계산식, 파싱 규칙, 검수 기준을 바꿀 때는 코드를 수정하기 전에 이 문서를 먼저 갱신합니다.

## 문서 목록

- `cost-db-rules.md`
  - Cost DB와 참조 워크북 해석 규칙
- `reference-workbook-inspection.md`
  - 업로드 검수 모달에서 확인해야 할 항목과 경고 규칙
- `estimate-calculation-rules.md`
  - 견적 계산 순서, 물가상승, 마진, 하자보수, 비정형 문서 반영 금지 원칙
- `economics-rules.md`
  - 경제성 계산, 스택 교체비, 민감도, 격자 분석 규칙
- `change-checklist.md`
  - 변경 전후 공통 확인 절차

## 사용 순서

1. 변경 요청이 들어오면 먼저 관련 spec 문서를 확인합니다.
2. 기존 문서와 구현이 충돌하면 문서를 기준으로 판단합니다.
3. 계산 로직 변경이면 `estimate-calculation-rules.md`를 먼저 수정합니다.
4. 파싱/검수 변경이면 `cost-db-rules.md`, `reference-workbook-inspection.md`를 같이 확인합니다.
5. 구현 후에는 `change-checklist.md` 기준으로 검증합니다.
