# 변경 체크리스트

## 목적

기능을 추가하거나 수정할 때 계산, 파싱, 화면 규칙이 빠지지 않도록 최소 점검 순서를 고정합니다.

## 1. 변경 시작 전

- 이번 작업이 어떤 영역인지 분류했습니다.
  - Cost DB / 참조 워크북
  - 견적 계산
  - 경제성
  - B2B 제안
  - 공통 UI
- 관련 `specs/` 문서를 먼저 읽었습니다.
- 공식이나 파싱 규칙이 바뀌면 문서를 먼저 수정했습니다.

## 2. 구현 중

- 계산식은 `lib/estimator.ts` 또는 `lib/economics.ts` 한 곳에서만 수정했습니다.
- 화면 컴포넌트가 계산식을 중복 구현하지 않도록 유지했습니다.
- 금액 단위를 `억원` 기준으로 확인했습니다.
- 영어 라벨이 새로 들어가면 한국어로 바꿀 수 있는지 먼저 검토했습니다.

## 3. 참조 워크북 관련 변경

- 업로드 후 즉시 반영되지 않고 검수 모달이 먼저 뜨는지 확인했습니다.
- 검수 모달에서 아래 항목이 수정 가능한지 확인했습니다.
  - 프로젝트명
  - 기준연도
  - 기준용량
  - 항목명
  - S/P/C 분류
  - 금액
- 항목 수정 후 총액, S/P/C 소계, 경고가 즉시 다시 계산되는지 확인했습니다.

## 4. 견적산출 관련 변경

- 용량 환산 기준금액이 정상인지 확인했습니다.
- 물가상승 반영 기준금액이 정상인지 확인했습니다.
- 현장 가산, 도면 차이 가산, 하자보수 합계가 정상인지 확인했습니다.
- 리스크 시나리오와 입찰 전략 카드가 깨지지 않는지 확인했습니다.

## 5. 경제성 관련 변경

- 시장 전광판이 `/economics`에만 보이는지 확인했습니다.
- `/estimate`에는 전광판이 없는지 확인했습니다.
- LCOE, Project IRR, Equity IRR, DSCR 계산 결과가 렌더링되는지 확인했습니다.
- 민감도 분석과 토네이도 차트가 깨지지 않는지 확인했습니다.
- 스택 교체 주기/교체비 입력이 연간 현금흐름에 반영되는지 확인했습니다.
- 판매단가/연료비 격자 분석이 렌더링되는지 확인했습니다.

## 6. 보안 관련 변경

- 업로드 파일 처리 안내 문구가 화면에 보이는지 확인했습니다.
- `layout.tsx`에 `referrer=no-referrer` 설정이 반영되었는지 확인했습니다.
- 지도/시세 외 외부 호출이 없는지 확인했습니다.
- 숨겨진 시트/메모 감지 경고가 검수 흐름에 반영되는지 확인했습니다.

## 7. 최종 확인

- `npm run build` 통과
- `/estimate` 확인
- `/economics` 확인
- `/b2b` 확인
- 숫자 표기 확인
  - 금액: `억원`
  - 비율: `%`
  - 용량: `MW`

## 8. 문서 갱신 여부

- `AGENTS.md` 반영 필요 여부 확인
- `specs/README.md` 반영 필요 여부 확인
- 관련 스펙 문서 반영 여부 확인
