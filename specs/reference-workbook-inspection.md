# 참조 워크북 검수 규칙

## 목적

엑셀 업로드 직후 자동 반영으로 인해 잘못된 총액이 견적에 들어가는 문제를 막기 위한 검수 흐름을 정의합니다.

## 기본 흐름

1. 사용자가 기준 워크북을 업로드합니다.
2. 브라우저에서 로컬로 파싱합니다.
3. 즉시 반영하지 않고 검수 모달을 엽니다.
4. 사용자가 내용을 확인하고 수정합니다.
5. `검수 후 반영`을 눌렀을 때만 기준 데이터로 저장합니다.

## 검수 모달에서 확인할 항목

- 파일명
- 기준 프로젝트명
- 기준연도
- 기준용량(MW)
- 파싱 총액(억원)
- 항목 수
- 읽은 시트 수
- S/P/C 카테고리 소계
- 항목별 상세 테이블

## 항목별 편집 기능

- 항목명 수정
- S/P/C 분류 변경
- 금액(억원) 수정
- 특정 항목 제외

위 편집이 일어나면 아래 값은 즉시 다시 계산되어야 합니다.

- 총액
- 항목 수
- S/P/C 소계
- 경고 메시지

## 경고 규칙

아래 경우 경고를 표시합니다.

- 기준연도를 찾지 못함
- 기준용량을 찾지 못함
- 항목 수가 너무 적음
- 총액이 30억원 미만
- 총액이 2,000억원 초과
- 특정 카테고리 비중이 85% 이상
- S/P/C 중 비어 있는 카테고리 존재

## 반영 규칙

- `취소`: 이번 업로드는 버림
- `검수 후 반영`: 현재 모달 상태 그대로 저장
- 모든 항목이 제외되면 반영 대상이 없어야 하며 저장하지 않습니다.

## 구현 책임

- 파싱과 검수 요약 계산: `lib/excel-import.ts`
- 검수 모달 UI: `components/estimate-studio.tsx`

## 실무 메모

- 검수 모달은 보기용이 아니라 수정용이어야 합니다.
- 숫자가 이상할 때 원본 엑셀을 다시 열기 전에 화면에서 바로 조정할 수 있어야 합니다.
