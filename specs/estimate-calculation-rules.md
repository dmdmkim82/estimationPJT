# 견적 계산 규칙

## 목적

견적산출 페이지에서 총액이 어떤 순서와 공식으로 계산되는지 고정합니다.

이 문서는 계산 정합성을 위한 기준 문서입니다.

## 최상위 정책

### 결정론적 계산

최종 견적 계산은 반드시 로컬 계산식으로 처리합니다.

- 동일한 입력값이면 동일한 결과가 나와야 합니다.
- 모델 버전, 프롬프트, 외부 응답 상태에 따라 금액이 달라지면 안 됩니다.

### 비정형 문서 자동 반영 금지

현장조사서, 계약서, 회의록, PDF/Word 문서의 비정형 문장은 자동으로 견적 금액에 반영하지 않습니다.

허용:

- 후보 항목 추출
- 검토 포인트 제시
- 구조화 입력 초안 생성

금지:

- 문장에서 읽은 내용을 바로 금액 계산에 투입
- 사용자가 확인하지 않은 수량/단가 자동 확정
- “AI가 그렇게 판단했다”는 이유만으로 가산 항목 반영

즉, 계산 엔진에 들어가는 값은 아래 중 하나여야 합니다.

- 기준 워크북에서 검수 후 확정된 값
- 사용자가 직접 입력한 값
- 시스템에 정의된 고정 계산식 결과

## 입력값

- 기준 프로젝트
- 목표 프로젝트 용량(MW)
- 프로젝트 유형
  - 단층 옥외형
  - 단층 + 사무동
  - 복층형
- 기준연도
- 착공연도
- S/P/C 물가상승률
- 현장 가산 입력
- 도면 차이 입력
- 목표 마진율
- 하자보수율

## 계산 순서

### 1. 기준 프로젝트 용량 스케일링

기준 프로젝트 항목 금액을 목표 용량으로 비례 환산합니다.

- 스케일 계수:
  - `목표 용량 / 기준 프로젝트 용량`

### 2. 항목별 물가상승 반영

- `years = max(0, 착공연도 - 기준연도)`
- 카테고리별 복리 계산:
  - `S = amount × (1 + serviceRate)^years`
  - `P = amount × (1 + procurementRate)^years`
  - `C = amount × (1 + constructionRate)^years`

### 3. 프로젝트 유형 가산

- 단층 + 사무동
  - 사무동 / 철골
  - HVAC 및 부대설비
- 복층형
  - 복층 구조 프레임
  - 수직 이송 설비

추가 가산은 MW 또는 기준값 기준으로 계산하고, 동일하게 물가상승을 적용합니다.

### 4. 도면 차이 가산

- 토목 변경률
- 전기 변경률
- 기계 변경률
- 제어/계장 변경률

기존 카테고리 금액을 베이스로 비율 가산 후 물가상승을 적용합니다.

### 5. 현장 가산

현장 검토 입력값을 기준으로 별도 항목을 추가합니다.

예시:

- 부지 정지
- 옹벽
- HRU
- 온수 시스템
- 중온수 시스템
- 가공선로
- 지중선로
- 열배관

### 6. EPC 소계

- `costSubtotal = 기준 항목 + 프로젝트 유형 가산 + 도면 차이 가산`

### 7. 공사비 원가 소계

- `quotedBase = costSubtotal + siteSubtotal`

### 8. 마진 반영 공사비

- `constructionQuote = quotedBase / (1 - marginRate)`

여기서 `marginRate`는 `%` 입력값을 소수로 변환해 사용합니다.

### 9. 하자보수

- `warranty = constructionQuote × warrantyRate`

### 10. 총 최종 견적가

- `grandTotal = constructionQuote + warranty`

## 결과 화면에서 반드시 보여줄 값

- 총 최종 견적가
- 용량 환산 기준금액
- 물가상승 반영 기준금액
- 핵심 EPC 소계
- 현장 가산
- 도면 차이 가산
- 하자보수
- S/P/C 소계

## 리스크 시나리오

최소 아래 4개는 유지합니다.

- 복합 물가상승
- 공기 지연
- 계통/토목 추가 리스크
- 조달 원자재 리스크

## 구현 책임

- 견적 계산 로컬 엔진: `lib/estimator.ts`
- 화면 표시 및 입력 조합: `components/estimate-studio.tsx`

## 변경 시 주의사항

- 공식 변경 전 이 문서를 먼저 수정합니다.
- `formatEok`, `formatPercent` 출력 단위를 임의로 바꾸지 않습니다.
- 계산식이 바뀌면 리스크 시나리오와 기준 비교 로직도 같이 검토합니다.
- 비정형 문서 기능을 붙이더라도, 최종 계산 입력값은 반드시 구조화/확정 단계 이후에만 반영합니다.
