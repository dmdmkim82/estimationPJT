# Cost DB 규칙

## 목적

참조 워크북과 Cost DB 엑셀을 읽을 때 어떤 값을 기준으로 견적에 반영하는지 명확히 정의합니다.

## 기본 원칙

- 기준 프로젝트는 `ReferenceProject` 구조로 정리합니다.
- 기준 프로젝트의 핵심 속성은 아래 5개입니다.
  - 프로젝트명
  - 기준연도
  - 기준용량(MW)
  - 총액(억원)
  - 항목 목록(`items`)
- 항목 목록은 아래 필드를 가집니다.
  - `code`
  - `name`
  - `category`
  - `amountEok`
  - `note`

## 카테고리 규칙

- `S`: 서비스, 엔지니어링, 설계, 인허가, 시운전, PM
- `P`: 조달, 장비, 기자재, 패키지, 전기 패키지, 구매
- `C`: 시공, 설치, 토목, 기계 설치, 배관, 건축, 철골

카테고리 텍스트가 없으면 항목명과 비고를 바탕으로 추론합니다.

## 금액 단위 규칙

- 앱 내부 기준 금액 단위는 `억원`입니다.
- 워크북 파싱 시 아래 힌트를 우선 사용합니다.
  - 헤더에 `억원`, `eok`, `100m`가 있으면 그대로 사용
  - `백만원`, `million`이면 `억원`으로 환산
  - `원`, `krw`이면 `억원`으로 환산
- 문맥으로 단위를 알 수 없을 때는 숫자 크기로 보정하되, 검수 모달에서 반드시 다시 확인합니다.

## 파싱 대상 행 규칙

- 항목명 없음: 제외
- 금액 없음 또는 0: 제외
- `합계`, `총계`, `subtotal`, `total`: 제외

## 기준 프로젝트 반영 규칙

- 기본 내장 기준 프로젝트는 샘플 기준값입니다.
- 업로드한 워크북 기준 프로젝트가 있으면 사용자가 검수 후 우선 반영합니다.
- 업로드된 기준 프로젝트는 localStorage에 저장합니다.

## 실무 주의사항

- 기준연도와 기준용량은 파일명이나 시트 내용에서 추정되므로 항상 검수합니다.
- S/P/C 한 카테고리에 과도하게 치우치면 금액 열 또는 분류 추론이 잘못되었을 가능성이 큽니다.
- 참조 총액이 비정상적으로 크거나 작으면 단위 환산부터 의심합니다.
